# SUB-31 Implementation Evidence: Immutable Artifacts

Status: Approved

## Scope

- Original and generated artifact bytes stored in MinIO.
- PostgreSQL metadata records SHA-256, byte size, media type, filename, kind, actor, organization, contract, agency, object key, and timestamp.
- Replacement and regeneration create new artifact/object identities plus immutable predecessor relations.
- Download resolves metadata, enforces role/organization/submission scope, reads the referenced object, and verifies size and SHA-256 before returning bytes.

## Integrity And Authority Boundaries

- Object keys include organization, unique artifact identity, and content digest and are never reused by the application.
- PostgreSQL triggers reject metadata/relation update or deletion.
- Authorization occurs before MinIO retrieval, including indirect-reference attempts.
- Auditor access is read-only; government access remains gated on submitted state.

## Verification

- Store/read real original and generated bytes.
- Verify complete metadata and MinIO reference.
- Preserve predecessor bytes after replacement and record relation.
- Reject direct record deletion.
- Deny cross-organization download before contacting object storage.
